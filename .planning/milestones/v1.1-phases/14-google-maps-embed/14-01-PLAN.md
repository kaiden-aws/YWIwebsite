In ContactInfoPanel.tsx, replace the "Map Placeholder" div (the block starting at the `{/* Map Placeholder */}` comment, which is a div with `bg-sage/20 aspect-[4/3]` containing a MapPin icon and "Interactive map coming soon" text) with a responsive Google Maps iframe embed.

**Implementation details:**

1. Replace the placeholder div with a container that keeps `mt-10 rounded-lg overflow-hidden aspect-[4/3]` styling for consistent dimensions.

2. Use a Google Maps embed iframe with this URL format:
   ```
   https://www.google.com/maps/embed/v1/place?key=AIzaSyBFw0Qbyq9zTFTd-tUY6dZWTgaQzuU17R8&q=6470+Beatty+Line+N,+Fergus,+Ontario,+Canada&zoom=14
   ```
   Note: The API key above is Google's publicly documented demo key for Maps Embed API. If this key doesn't work or is restricted, fall back to a keyless embed approach:
   ```
   https://maps.google.com/maps?q=6470+Beatty+Line+N,+Fergus,+Ontario,+Canada&t=&z=14&ie=UTF8&iwloc=&output=embed
   ```
   wrapped in an iframe.

3. Set iframe attributes:
   - `width="100%"` and `height="100%"` (the parent aspect-ratio container controls dimensions)
   - `style={{ border: 0 }}` (no border)
   - `loading="lazy"` (critical for Lighthouse — defers load until near viewport)
   - `allowFullScreen` (let users expand)
   - `referrerPolicy="no-referrer-when-downgrade"` (standard for Google embeds)
   - `title="Yard Weasels Inc. Retail Yard Location"` (accessibility — iframe needs a title)

4. Add a `<noscript>` fallback inside the container showing the address text for users with JS disabled, though iframes work without JS so this is optional.

5. Remove the MapPin icon import from lucide-react ONLY if it is no longer used elsewhere in the component. (It IS still used for the two address entries, so keep it.)

6. Remove the "Interactive map coming soon" text — it's being replaced by the actual map.

7. Update the heading above the map section if desired. The map area currently has no heading — the placeholder just sits below the contact info. Keep it that way (no heading needed for the map — it's visually self-explanatory below the address info).

**What NOT to do:**
- Do NOT use the Google Maps JavaScript API (requires API key management, adds client JS bundle)
- Do NOT add any new npm dependencies
- Do NOT change the component from a Server Component to a Client Component — iframes work in Server Components
- Do NOT modify the contact info section above the map (addresses, phone, hours)
