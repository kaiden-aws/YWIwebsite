    2. Add `import Image from 'next/image'`
    3. Replace the `<ImagePlaceholder label={project.image} className="!aspect-auto h-full w-full !rounded-none" />` with:
       ```tsx
       <Image

         fill

         className="object-cover"
       />
       ```
       The parent m.div already has `relative overflow-hidden` and fixed row heights via `auto-rows-[200px] md:auto-rows-[250px]`, so Image fill will stretch to cover the grid cell. No additional wrapper div needed. `sizes` matches the 2-col mobile / 3-col desktop grid. Use `project.name` as alt text (descriptive) instead of `project.image` (a file path).

    **GalleryLightbox.tsx:**
    1. Remove `import ImagePlaceholder from '@/components/ui/ImagePlaceholder'`
    2. Add `import Image from 'next/image'`
    3. Replace the `<ImagePlaceholder label={current.image} aspectRatio="video" className="!rounded-lg w-full" />` with:
       ```tsx
       <div className="relative aspect-video w-full overflow-hidden rounded-lg">
         <Image
           src={current.image}
           alt={current.name}
           fill
           sizes="(max-width: 768px) 100vw, 80vw"
           className="object-cover"
         />
       </div>
       ```
       A wrapper div is needed here because the lightbox content container is not a fixed-dimension element — `aspect-video` provides the height reference for fill mode. `sizes="(max-width: 768px) 100vw, 80vw"` matches the max-w-5xl container with mx-4 padding. Use `current.name` as alt text.

    **ProjectShowcase.tsx:**

       ```tsx
           alt={project.name}
           fill
           sizes="(max-width: 768px) 50vw, 33vw"
           className="object-cover"
         />
       </div>
       ```
       The wrapper div provides `aspect-square` to maintain the same square proportion that was set via ImagePlaceholder's aspectRatio="square" prop. `sizes` matches the 2-col mobile / 3-col desktop grid. Use `project.name` as alt text.
