<task type="auto">
  <name>Task 2: Add JavaScript minimum depth validation (0.5 inches) to MaterialCalculator</name>
  <files>src/components/sections/products/MaterialCalculator.tsx</files>
  <action>
    In the `validate()` function, update the depth validation block. Currently the depth check (around line 63) only rejects `num <= 0`. Add an explicit check for the 0.5-inch minimum AFTER the `<= 0` check and BEFORE the `> 36` max check:

    ```ts
    const depthVal = form.depth.trim()
    if (!depthVal) {
      newErrors.depth = 'Depth is required'
    } else {
      const num = parseFloat(depthVal)
      if (isNaN(num) || num <= 0) {
        newErrors.depth = 'Enter a valid positive number'
      } else if (num < 0.5) {
        newErrors.depth = 'Minimum depth is 0.5 inches'
      } else if (num > 36) {
        newErrors.depth = 'Maximum depth is 36 inches'
      }
    }
    ```

    The key change is adding the `else if (num < 0.5)` branch with a clear user-facing error message. The HTML `min={0.5}` attribute already exists on the input element but browsers allow bypassing it (typing values manually, pasting, programmatic input). This JavaScript validation ensures the minimum is enforced regardless of how the value is entered.

    Do NOT change the length or width validation, the calculation logic, the HTML attributes, or any styling.
  </action>
  <verify>

  </verify>
  <done>MaterialCalculator rejects depth values below 0.5 inches via JavaScript validation with a user-visible error message ("Minimum depth is 0.5 inches"), not relying solely on the HTML min attribute.</done>
