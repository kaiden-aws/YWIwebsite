  <files>src/components/sections/products/MaterialCalculator.tsx</files>
  <action>
Create `MaterialCalculator.tsx` as a `'use client'` component. This is the ONLY client component introduced in Phase 6.

**State management** — Use `useState` (no form library):
- `form: { length: string, width: string, depth: string, productType: string }` — store numeric inputs as STRINGS to avoid NaN/0 display issues when user clears input fields. Initialize productType to `products[0].id`.
- `errors: { length?: string, width?: string, depth?: string }` — inline error messages per field
- `result: number | null` — calculated cubic yards, null when no calculation done or after validation failure

**Imports:**
- `'use client'` directive at top
- `useState` from `react`
- `Link` from `next/link`
- `{ products }` from `@/lib/data/products`
- `{ cn }` from `@/lib/utils/cn`

**Validation function** `validate()`:
- Length: required, must parse to positive number, range 0.1-1000 ft. Error messages: "Length is required", "Enter a valid positive number", "Maximum length is 1,000 ft"
- Width: same pattern as length with appropriate labels
- Depth: required, must parse to positive number, range 0.5-36 inches. Error messages: "Depth is required", "Enter a valid positive number", "Maximum depth is 36 inches"
- Use `.trim()` on the string value before checking emptiness
- Return a `FormErrors` object — empty object means valid

**handleCalculate function:**
- Call `validate()`, set errors
- If any errors exist, call `setResult(null)` to clear stale results, return early
- Parse all three inputs to `parseFloat()`
- Formula: `(length * width * (depth / 12)) / 27` = cubic yards
- Round to 2 decimal places: `Math.round(cubicYards * 100) / 100`
- Set result

**Layout:**
- Section wrapper: `py-20 md:py-28 px-6 bg-white`
- Container: `max-w-3xl mx-auto`
- Heading: `h2` "Material Calculator" in `font-display text-3xl md:text-4xl text-forest text-center mb-4`
- Subtitle: "Estimate how much material you need for your project." in `text-charcoal/70 text-center`
- Input grid: `grid grid-cols-1 sm:grid-cols-3 gap-6 mb-6` for the 3 numeric inputs
- Each input: `label` with `htmlFor`, `input type="number"` with `inputMode="decimal"`, styled with `cn()` for error/normal states (border-red-500 on error, border-charcoal/20 normally), `placeholder` (e.g. "e.g. 20"), `min`/`max`/`step` attributes, `aria-describedby` linking to error `p` element, `aria-invalid` when error exists
- Error text: `p` with `role="alert"`, `text-sm text-red-600`, `mt-1`
- Product type dropdown: `select` element below the input grid, `mb-8`, styled to match inputs. Populate with `products.map(p => <option key={p.id} value={p.id}>{p.name}</option>)`. Label: "Product Type"
- Calculate button: `button` with `onClick={handleCalculate}`, `w-full sm:w-auto px-8 py-3 bg-forest hover:bg-forest-light text-cream font-medium rounded-lg transition-colors`
- Result area (only shown when `result !== null`): `mt-8 p-6 bg-sage/10 rounded-lg text-center` with:
  - "You need approximately" in `text-charcoal/70`
  - `{result} cubic yards` in `font-display text-3xl md:text-4xl text-forest`
  - "of {selectedProduct?.name}" in `text-charcoal/60`
  - "Request Delivery Quote" `Link` to `/contact` in `inline-flex items-center px-8 py-4 bg-terracotta hover:bg-terracotta-light text-cream font-medium rounded-lg transition-colors mt-6`

**Critical implementation notes:**
- Store inputs as strings, parse to float ONLY during validation/calculation
- Clear result (`setResult(null)`) when validation fails — never show stale result alongside errors
- The product type dropdown does NOT change the formula — it only labels the result text
- Use `id` attributes on inputs and `htmlFor` on labels for accessibility
