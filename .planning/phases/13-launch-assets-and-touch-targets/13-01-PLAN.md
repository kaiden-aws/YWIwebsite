    **Step 1: Create shared OG image utility** at `src/lib/og-image.tsx`.

    Export a function `generateOGImage({ title, subtitle }: { title: string; subtitle?: string })` that returns an `ImageResponse` from `next/og`.

    The image design (1200x630px):
    - Full background: forest color `#1a3a2a`
    - A horizontal terracotta accent bar (`#c4703f`) at the top, 6px tall
    - Company name "YARD WEASELS INC." at top in cream (`#f5f0e8`), 32px, uppercase, letter-spacing 4px
    - Page title in cream, 64px bold, centered vertically
    - Optional subtitle below title in cream at 60% opacity, 28px
    - Bottom text: "yardweasels.ca" in cream at 40% opacity, 24px
    - Use inline styles (JSX with `style={{}}`) since ImageResponse uses Satori which supports a subset of CSS (no Tailwind). Use `display: 'flex'` on all containers (Satori requires explicit flex).

    Also export `const size = { width: 1200, height: 630 }` and `const contentType = 'image/png'`.

    **Step 2: Create per-page opengraph-image.tsx files** in each of the 6 route directories.

    Each file follows this pattern:
    ```tsx
    import { generateOGImage, size, contentType } from '@/lib/og-image'
    export { size, contentType }
    export default function Image() {
      return generateOGImage({ title: 'Page Title', subtitle: 'Optional subtitle' })
    }
    ```

    Page-specific titles and subtitles:
    - Homepage (`src/app/opengraph-image.tsx`): title "Professional Landscaping", subtitle "Fergus, Ontario & Centre Wellington"
    - About (`src/app/about/opengraph-image.tsx`): title "About Our Team", subtitle "Rooted in Fergus, Ontario"
    - Services (`src/app/services/opengraph-image.tsx`): title "Our Services", subtitle "Design, Build, Maintain & More"
    - Products (`src/app/products/opengraph-image.tsx`): title "Quality Materials", subtitle "Aggregates, Mulch, Stone & More"
    - Gallery (`src/app/gallery/opengraph-image.tsx`): title "Project Gallery", subtitle "Our Work Across Centre Wellington"
    - Contact (`src/app/contact/opengraph-image.tsx`): title "Contact Us", subtitle "Get a Free Landscaping Quote"

    **Step 3: Remove manual `images` arrays from page metadata exports.**

    The `opengraph-image.tsx` convention automatically injects the correct OG image meta tags, so the manual `images: [{ url: '/og-*.jpg', ... }]` arrays in each page's `openGraph` metadata are now redundant and reference nonexistent files. Remove the `images` property from the `openGraph` object in all 6 page.tsx files. Keep the rest of the openGraph properties (`title`, `url`, `description`, spread of `sharedOpenGraph`).

    Also remove the `ogImage` property from `siteConfig` in `src/lib/metadata.ts` and the fallback `images` array from the root layout's `openGraph` in `src/app/layout.tsx` â€” these are superseded by the convention-based files.

    **Important notes:**
    - `ImageResponse` is imported from `next/og` (built into Next.js 16, no extra package needed)
    - Satori (the renderer behind ImageResponse) does NOT support Tailwind classes, only inline `style` objects
    - All container divs in the JSX MUST have `display: 'flex'` set explicitly
    - Font rendering: use system fonts (the `style` prop supports `fontFamily` with web-safe fonts). Use sans-serif for body text and serif for the title to approximate the site's DM Serif Display / Plus Jakarta Sans pairing.

    With `text-sm` (~20px line-height) and `py-2.5` (10px top + 10px bottom), total height is ~40px â€” below the 44px WCAG touch target minimum.

    Fix: Add `min-h-[44px] inline-flex items-center` to the CTA Link. This ensures the button always renders at least 44px tall while maintaining the current visual padding. The `inline-flex items-center` keeps the text vertically centered within the taller hit area.
