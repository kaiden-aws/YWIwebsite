**Fix 1 — GalleryLightbox.tsx:**
Add `'use client'` as the very first line of the file, before any imports. The file uses `useEffect`, `useRef`, `useCallback` hooks and Framer Motion `m` components — it must be explicitly marked as a client component. Currently it works only because GalleryGrid.tsx (its parent) has `'use client'`, but this is fragile. Every other hook-using component in the project (HeroParallax, TestimonialCarousel, MaterialCalculator, GalleryGrid) already has `'use client'`. This makes GalleryLightbox consistent.

The file after the directive:
```
'use client'

import { useEffect, useRef, useCallback } from 'react'
import { AnimatePresence, m } from 'motion/react'
// ... rest unchanged
```

**Fix 2 — Footer.tsx:**
Replace the hardcoded `const services = [...]` array (lines 4-11) with an import from the canonical data source. The current hardcoded list has a label mismatch: `'Irrigation & Lighting'` vs the canonical `'Irrigation & Landscape Lighting'` in services.ts.

Steps:
1. Add import: `import { services } from '@/lib/data/services'` alongside the existing navigation import
2. Remove the entire `const services = [...]` block (6 hardcoded strings)
3. In the JSX services `.map()`, change from `{services.map((service) => ( <li key={service}> ... {service} ...` to `{services.map((service) => ( <li key={service.id}> ... {service.title} ...` — because `services.ts` exports `Service[]` objects with `id` and `title` properties, not plain strings

CRITICAL: Do NOT add `'use client'` to Footer.tsx. The `services.ts` file exports plain objects with no hooks or browser APIs. Footer remains a pure Server Component.
